Abstract

This report presents a systematic analysis of five Linux kernel local privilege escalation (LPE) vulnerabilities spanning 2016 to 2026. We identify a structural evolutionary pattern: the attack surface migrated from the Virtual Memory subsystem through File System pipes to the Crypto API and Networking stacks. Exploit reliability improved from probabilistic race conditions to fully deterministic cross-distribution primitives. A Docker-based eBPF-instrumented testbed characterizes each vulnerability's page-cache corruption primitive and evaluates mitigation effectiveness.

Index Terms: Linux kernel security, LPE, page-cache corruption, CVE analysis, Docker, eBPF, AI-assisted vulnerability discovery, container escape.

CVE Analysis

Dirty COW
CVE-2016-5195
2016 CVSS 7.8 Race Condition
Subsystem: Virtual Memory (mm/memory.c)
Primitive: Race window → COW bypass → page write
Reliability: ~30% (slow, can crash system)
Container Escape: No
Discovery: Manual, 9-year-old bug

Exploited a race in /proc/self/mem copy-on-write path. Notorious for unreliability — required multiple retries and risked kernel panic.
Dirty Pipe
CVE-2022-0847
2022 CVSS 7.8 Race-Free
Subsystem: File System (fs/pipe.c)
Primitive: Uninit PIPE_BUF_FLAG_CAN_MERGE → page overwrite
Reliability: 100% deterministic
Container Escape: No
Discovery: Manual pipe buffer audit

First fully deterministic Linux LPE. Version-specific to kernel 5.8+. Achieved root by injecting into pipe's page-cache entry via splice().
Copy Fail
CVE-2026-31431
2026 CVSS 7.8 Deterministic Container Escape
Subsystem: Crypto API — algif_aead (AF_ALG)
Primitive: Logic flaw → 4-byte page-cache write
Reliability: 100% — single 732-byte Python script
All distros since: 2017
Discovery: AI (Xint Code) — ~1 hour

A straight-line logic bug in the kernel crypto API. No race, no offsets. Universal across Ubuntu, RHEL, Amazon Linux, SUSE. AI-discovered.
Dirty Frag
CVE-2026-43284 / 43500
2026 CVSS 7.8 Deterministic Container Escape
Subsystem: Networking — IPsec ESP + RxRPC (XFRM)
Primitive: Page frag CoW bypass via ESP+RxRPC chain
Reliability: 100% — no race, no offsets
Discovery: Manual (Hyunwoo Kim)

Chains two CVEs in the networking stack. First LPE via IPsec subsystem. Affects ESP4/ESP6 + RxRPC modules. Also known as "Copy Fail 2".
Fragnesia
CVE-2026-46300
2026 CVSS 7.8 Deterministic Container Escape
Subsystem: Networking — XFRM ESP-in-TCP (skb_try_coalesce)
Primitive: Missing SKBFL_SHARED_FRAG → page-cache write
Reliability: 100% — PoC overwrites /usr/bin/su
Discovery: AI (Zellic AI tool, William Bowling)

Accidentally introduced by the Dirty Frag patch. The Dirty Frag fix exposed a latent bug. Demonstrates patches as attack surface.

Key Findings

F1

Reliability Evolution

From ~30% (Dirty COW) to 100% deterministic (all 2026 CVEs). Race conditions are no longer required.

F2

Subsystem Migration

Attack surface: VM → FS → Crypto → Networking. Each move exploits subsystems with less security scrutiny.

F3

Unified Primitive

All five CVEs share page-cache corruption. A unified eBPF LSM hook could detect all variants.

F4

AI Acceleration

Copy Fail found in ~1 hour by AI (Xint Code). Fragnesia by Zellic AI. Manual discovery: weeks.

F5

Patch-Induced Exposure

Fragnesia was accidentally activated by the Dirty Frag patch — security fixes must be adversarially reviewed.

F6

Container Threat

All 2026 CVEs enable container escape — Kubernetes, CI/CD, and multi-tenant environments are at direct risk.

F7

Rapid Patching

Patch turnaround collapsed to 0–1 days for 2026 CVEs, reflecting matured kernel response infrastructure.

F8

Mitigation Gap

SMEP/SMAP and SELinux are ineffective against 2026 CVEs. Module blacklisting is the only practical interim fix.

Docker Research Testbed

The research testbed runs each CVE in an isolated, network-restricted container. No real exploitation occurs — all containers simulate analysis in userspace with pinned kernel context.

Container Safety Model

All containers: cap_drop: ALL · no-new-privileges · read_only: true · internal network only (no egress). eBPF tracer container requires privileged for kernel probe access only.
